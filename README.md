# MXTP - Mixtape CLI Tools

[![Tests](https://woodpecker.carlosmolero.com/api/badges/5/status.svg)](https://woodpecker.carlosmolero.com/repos/5)

**MXTP** is a command-line tool to prepare and organize mixtapes for cassette recording with maximum quality. It helps you minimize distortion, optimize space, and balance track distribution between Side A and Side B.

## Requirements

- Bash 5.\*
- ffmpeg
- auto-editor
- gum
- jq
- bc

## Installation

```bash
brew tap devcarlosmolero/homebrew https://codeberg.org/devcarlosmolero/homebrew
brew install devcarlosmolero/homebrew/mxtp
```

## Usage

MXTP processes your audio files through 4 fundamental commands. These commands are executed in a \_**predefined sequential order**, ensuring a consistent workflow regardless of the order in which they are called.

> [!TIP]
> For detailed usage instructions and available flags, run `mxtp help` or `mxtp prepare -h`.

### Commands overview

| Command        | Description                                                                                                                                                                         |
| :------------- | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Duration**   | Calculates and displays the total duration of all `.mp3` files within the target directory.                                                                                         |
| **Trim**       | Automatically removes silences using `auto-editor` to save precious space on your cassettes.                                                                                        |
| **Normalize**  | Adjusts audio peaks using `ffmpeg` to prevent distortion and ensure high-quality recording levels.                                                                                  |
| **Reorganize** | Optimizes cassette space based on available capacity (46, 60, or 90 mins). It uses a balancing algorithm to distribute tracks between Side A and Side B for an even storage layout. |

### Sequential export: The `-m` Flag

The `-m <path>` flag is a critical feature of MXTP. It allows you to move your processed files to an external volume or a specific directory **sequentially**.

> [!INFO]
> Many hardware devices and older filesystems prioritize the **file creation order** over alphabetical naming when determining the playback sequence.
> By using the `-m` flag, MXTP ensures that the specific playlist order generated by the `reorganize` command is physically preserved on the target storage, guaranteeing that your cassette is recorded exactly as planned.
